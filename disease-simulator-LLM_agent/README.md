# Overview

This repository is an extension of the infectious disease simulation described in ***"An Infectious Disease Spread Simulation Based on Large Language Model Decision Making"*** (Khaokaew et al., KDD 2026 AI4Science track).

It is forked from [RuochenKong/disease-simulator](https://github.com/RuochenKong/disease-simulator). The key change is replacing the logistic regression (LR) model for disease-reporting decisions with decisions generated by Large Language Models (LLMs), pre-computed into a **decision bank** and loaded at runtime.

# LLM-Based Reporting Rate

In the original simulation, an agent's probability of reporting influenza-like symptoms was computed via logistic regression over demographic attributes (age, race, gender, education, income). In this fork:

1. **Decision bank generation (offline):** For each unique demographic combination (encoded as a 5-digit key), an LLM is queried ≥5 times with a structured prompt asking whether the agent would self-report. Yes/No counts are stored in a CSV file.
2. **Decision bank loading (at startup):** The simulation reads the CSV and expands each row into a list of `true`/`false` values proportional to the Yes/No counts.
3. **Runtime decision:** When an infectious agent decides whether to report, it samples uniformly at random from its demographic group's decision list instead of drawing from a logistic regression probability.

### Decision bank CSV format

```
STRK,YES_C,NO_C
00100,44,64
00101,92,16
...
```

- `STRK` — 5-digit demographic key encoding `age | race | gender | education | income`
- `YES_C` / `NO_C` — LLM Yes/No response counts

### Key dimensions

The key is built by concatenating one digit per dimension listed in `biasConsideration` (in `bias.config`). Built-in dimensions:

| Name | Digits | Encoding |
|---|---|---|
| `Vulnerability` | 1 | `0` age < 50, `1` age ≥ 50 |
| `Race` | 1 | `1` White, `2` Black/Other, `3` Asian |
| `Gender` | 1 | `0` Female, `1` Male |
| `Income` | 1 | `0` ≤ $70k, `1` > $70k individual income |
| `Household Income` | 1 | `0` ≤ $100k, `1` > $100k household income |
| `Hispanic` | 1 | `0` No, `1` Yes |
| `Education` | 1 | `0` < HS, `1` HS/College, `2` Bachelor/Graduate |
| `Family` *(event)* | 1 | `0` infected+reported, `1` infected not reported, `2` none |
| `News` *(event)* | 1 | Region id mod 3 (proxy for regional news context) |
| `RegionCases` *(example)* | 1 | `0` none, `1` 1–9, `2` 10–100, `3` >100 new cases in agent's region today |
| `TotalCases` *(example)* | 1 | `0` none, `1` 1–9, `2` 10–100, `3` >100 new cases across whole simulation today |

To add a new dimension without recompiling: register it in `KeyDimensionRegistry.java`, add its name to `biasConsideration`, and regenerate the decision bank CSV.

### Example decision banks

Two example files are provided in `examples/`:

| File | Key format | Description |
|---|---|---|
| `llm.atl.csv` | 5-digit (`Vulnerability\|Race\|Gender\|Education\|Income`) | Pure demographic baseline |
| `llm.atl.S.key-event.csv` | 6-digit (demographic + 1 event dimension) | Demographic + event context |

# Biased Data Simulation

This simulation is built on the [*Patterns of Life simulation*](https://github.com/azufle/pol), a scalable agent-based model of human behavior. The disease spread and biased data generation features were introduced in Kong et al. (2024, 2025) and extended here with LLM-driven reporting decisions.

Extended features over the base Patterns of Life simulation:
- SEIR infectious disease model
- Real-world census-based population initialization (US census tracts)
- LLM decision bank for demographically-varying reporting rates
- Three simulation scenarios: independent reasoning, household influence, altruism/data-based message framing
- Single-bias and multivariate-bias reporting models (retained from upstream for comparison)

# How to Compile and Build

Import `pom.xml` as a Maven project. Four local JARs in `src/main/resources/libs/` must be installed into your local Maven repository before building:

```bash
mvn install:install-file -Dfile=src/main/resources/libs/jts-1.13.1.jar -DgroupId=com.vividsolutions -DartifactId=jts -Dversion=1.13.1 -Dpackaging=jar
mvn install:install-file -Dfile=src/main/resources/libs/geomason-1.5.2.jar -DgroupId=sim.util.geo -DartifactId=geomason -Dversion=1.5.2 -Dpackaging=jar
mvn install:install-file -Dfile=src/main/resources/libs/mason-19.jar -DgroupId=sim -DartifactId=mason -Dversion=19 -Dpackaging=jar
mvn install:install-file -Dfile=src/main/resources/libs/mason-tools-1.0.jar -DgroupId=at.granul -DartifactId=mason-tools -Dversion=1.0 -Dpackaging=jar
```

Build the executable fat JAR:

```bash
mvn org.apache.maven.plugins:maven-resources-plugin:2.6:resources \
  org.apache.maven.plugins:maven-compiler-plugin:3.1:compile \
  org.apache.maven.plugins:maven-assembly-plugin:3.1.0:single
```

Output: `target/vanilla-0.1-jar-with-dependencies.jar`

# How to Run a Simulation

## Option 1 — Maven (headless)

Use the provided scripts from the `disease-simulator-main/` directory. Both accept an optional log prefix and duration in days (defaults: `atl-test`, 90 days).

**Windows CMD:**
```bat
run.bat atl-test 90
```

**Git Bash / Linux / macOS:**
```bash
bash run.sh atl-test 90
```

> **Note:** Do not copy the Java command and run it directly in CMD or PowerShell. The `-Dlog4j2.configurationFactory=...` JVM flag contains a dot that Windows shells split into two tokens, causing the error `Could not find or load main class .configurationFactory=...`. The scripts handle this correctly by quoting each `-D` flag.

Alternatively, run the Java command directly on Linux / macOS:

```bash
java \
  "-Dlog4j2.configurationFactory=edu.gmu.mason.vanilla.log.CustomConfigurationFactory" \
  "-Dlog.rootDirectory=logs" \
  "-Dfile.prefix=atl-llm-s1" \
  "-Dsimulation.test=bias" \
  -jar target/vanilla-0.1-jar-with-dependencies.jar \
  -configuration examples/atlanta.properties \
  -bias.config examples/bias.llm.properties \
  -bias.single.config examples/bias.single.properties \
  -decision.bank examples/llm.atl.csv \
  -until 25920
```

**Scenario variants** — pass a different `-bias.config` and `-decision.bank` to the script, or edit `run.bat` / `run.sh` directly:

| Scenario | `-bias.config` | `-decision.bank` (example) |
|---|---|---|
| S1 Independent | `examples/bias.llm.properties` | `examples/llm.atl.csv` |
| S2 Household influence | `examples/bias.llm.S2.properties` | `examples/llm.atl.csv` |
| S3 Message framing | `examples/bias.llm.S3.properties` | `examples/llm.atl.csv` |

### CLI argument reference

| Argument | Description |
|---|---|
| `-Dlog.rootDirectory=<dir>` | Root directory for log output |
| `-Dfile.prefix=<name>` | Subdirectory name within log root |
| `-Dsimulation.test=bias` | Enable bias logging mode |
| `-configuration <file>` | Simulation parameters (agent count, SEIR params, map) |
| `-bias.config <file>` | Multivariate bias / LR coefficients |
| `-bias.single.config <file>` | Single-attribute bias parameters |
| `-decision.bank <file>` | LLM decision bank CSV **(new in this fork)** |
| `-until <steps>` | Stop after N steps (288 steps = 1 day at 5 min/step) |

Step conversion: 1 day = 288 steps, 30 days = 8640 steps, 90 days = 25920 steps.

# Set Model Parameters

Parameters are split across three configuration files:

- **`-configuration`** — Disease and simulation parameters. See `examples/atlanta.properties` for a full example.
- **`-bias.config`** — Multivariate bias model (logistic regression coefficients). Retained for comparison against LLM results. See `examples/bias.llm.properties`.
- **`-bias.single.config`** — Single-attribute reporting rates specified as age/income ranges. See `examples/bias.single.properties`.

Key disease parameters in the simulation config:

| Parameter | Description |
|---|---|
| `initPercentInfectious` | Initial infected population percentage |
| `exposedLasting` | Exposed phase duration range (days) |
| `infectiousLasting` | Infectious phase duration range (days) |
| `recoveredLasting` | Recovered phase duration range (days) |
| `additionalDiseaseSpreadingParam` | Extra spread probability per co-location |
| `probTestPositive` | Probability agent tests positive (default `0.05`) |
| `probClinicalDiagnosis` | Probability of clinical diagnosis (default `0.5`) |
| `reportingModel` | `LLM` (default) or `LR` — switches reporting decision method at runtime |

To run with the logistic regression model instead of the LLM decision bank, set `reportingModel = LR` in the simulation config. The `-decision.bank` argument is not required in LR mode.

# Load Maps

Default maps are in `src/main/resources/atlanta/`. Switch with the `maps` parameter:

```
maps = san-fran     # Partial San Francisco
maps = atlanta      # Atlanta downtown (default)
maps = gz-tianhe    # Tianhe, Guangzhou, China
```

Each map requires four ESRI shapefiles: `buildings`, `buildingUnits`, `walkways`, and `region_census`. See `documentaion/map.md` for details on obtaining and formatting these files.

Census attributes supported: Age (10 groups), Education (5 levels), Race (7 groups), Gender, Hispanic, Household Income (7 groups), Individual Income (6 groups).

# Output Dataset

Three output files are written to `logs/<prefix>/`:

- **`patterns_of_life.log`** — Console log; includes region-level agent summaries.
- **`AgentCharacteristicsTable.tsv`** — Per-agent demographics, `reportingRate` (LR probability), `reportingRates_Single`, and `keys_llm` (the 5-digit LLM lookup key).
- **`DiseaseReports.tsv`** — Per-infection events:

| Column | Description |
|---|---|
| `regionId` | Agent's home census tract |
| `diseaseStatus` | SEIR status at event time |
| `byAgentID` | Agent that transmitted the infection |
| `time` | Time of exposure |
| `location` / `checkin` | Geographic location and venue type |
| `Report(Component)` | Binary: reported via LLM decision bank |
| `Report(Single)` | Attribute types under which agent reported (e.g., `Age/EduLevel`) |

# Runtime and Scalability

For all paper experiments: **5,000 agents**, **30 simulation days** (~8,640 steps). Each run took approximately 5 hours on a single-threaded Intel i5-1135G7 at 2.40 GHz.

The simulation scales quadratically in agent count due to the underlying social network (more agents → more co-location pairs → quadratic increase in meetings).
