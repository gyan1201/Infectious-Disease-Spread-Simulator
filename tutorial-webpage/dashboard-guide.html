Dashboard UI Guide

A comprehensive walkthrough of the Streamlit interface, explaining how to tweak parameters, run agent-based models, and interpret the geographic disease reports.

⚙️ Tab 1: Configure & Run

The Configure & Run tab is the control center for your virtual pandemic. Here, you define the environment, human behavior models, and biological parameters of the disease before launching the Java backend.

🏙️ Scenario Settings

Define the physical rules and behavioral AI driving the agents.

  • City: Choose Atlanta or San Francisco. This changes the geographic boundaries, population density, and available locations for agents to travel to.
  • Reporting Model: Choose LLM (Large Language Model simulating complex human choices) or LR (Logistic Regression for basic probability).
  • Scenario (LLM only): Select behavioral biases. S1 (Independent) makes agents act purely on logic. S2 (Household) makes agents mimic family. S3 (Framing) changes how agents react to severe vs mild news.

🦠 Disease Parameters (SEIR)

Define the epidemiological rules of the virus.

  • Initial infectious (%): How much of the population starts with the virus on Day 0.
  • P(test positive | symptomatic): The probability an agent decides to get tested when showing symptoms. Lower values simulate testing scarcity.
  • P(clinical diagnosis): The probability a doctor correctly identifies the disease without a perfect lab test.
  • Simulation duration: How many days the virtual pandemic will last. Longer durations take longer to simulate.
Running the Simulation: Once you click the red "Run Simulation" button, the Streamlit server launches a background Java process. A yellow banner will appear at the top displaying the Process ID (PID). Wait for the green success message before moving to the Results tab.

📊 Tab 2: Results & Analysis

The Results tab visualizes the complex data generated by the simulation. It parses the DiseaseReports.tsv file into interactive geographic maps and time-series charts.

Step-by-Step UI Breakdown

1. The Data Selector

At the top of the tab, use the Select run dropdown to choose the data you want to view. By default, it will load the simulation you just finished (e.g., gui-run). You can also select the pre-computed examples/ to view instant results without waiting for a simulation.

2. The Time Machine (Day Navigation)

The simulation logs data in 5-minute increments, aggregated into days. Use the slider or the Next/Prev buttons to scrub through the timeline of the pandemic.

  • As you drag the slider, the Top KPI Metrics (Susceptible, Exposed, Infectious, Recovered) will instantly update to show the exact population stats for that specific day.
  • The map below will also dynamically recolor based on the day you select.

3. Interactive Geographic Maps

There are two primary map views you can toggle between using sub-tabs:

🗺️ SEIR Choropleth Map

This map shows the raw biological truth of the disease spread across different city regions.

Use the "Color by" dropdown to switch between viewing Infectious (Red gradient) or Susceptible (Blue gradient) agents. Hover your mouse over any neighborhood to see a popup tooltip containing the exact breakdown of agents in that specific region.

📢 Reporting Rate Map

This map shows the human behavior side of the simulation. It calculates the percentage of agents who actually reported their illness to the health system.

Formula: (Agents who reported / Total infected agents) * 100. Darker red indicates neighborhoods with higher compliance and testing rates.

4. The Pandemic Curve (Line Chart)

At the bottom of the screen (or to the right of the map on large screens), you will find the cumulative line chart. This visualizes the classic "Flatten the Curve" graph, tracking the total number of Exposed, Infectious, and Recovered agents over the entire duration of the simulation.
