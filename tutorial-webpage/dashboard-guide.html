Dashboard Usage Guide

A step-by-step guide on how to interact with the Python Streamlit simulation dashboard.

About the Dashboard

The dashboard allows you to simulate how a disease spreads through a city based on how people (agents) behave and make decisions. It interacts directly with the Java Agent-Based Modeling backend to run virtual pandemic scenarios.

1 Configure & Run a Simulation

When you first open the dashboard, you will be on the Configure & Run tab. This is where you set the rules for your virtual pandemic.

  • Pick a City: Choose either Atlanta or San Francisco. This changes the geographic map and the population density.
  • Choose a Scenario (LLM Bias): The simulation models different types of human behaviors:
    • S1 - Independent: Agents make decisions on their own.
    • S2 - Household Influence: Agents copy what their family members are doing.
    • S3 - Message Framing: Agents react differently based on how news/warnings are framed.
  • Adjust the Parameters: You can set the number of people in the city, how many days the simulation lasts, and how easily the disease spreads (SEIR parameters).
  • Click "Run Simulation": Hit the big red button! The server will start running the Java backend to simulate thousands of people interacting. Wait for the green success message.
Note: By default, the simulation saves the log prefix as gui-run. Remember this prefix for the next step!

2 View the Results

Once the simulation finishes running, click on the "Results" tab at the top of the page to analyze the geographic spread.

  • Select your run: In the dropdown menu, select gui-run (or whatever you named your log prefix) to load the data you just generated. You can also select one of the examples/ to see pre-generated data!
  • Use the Time Slider: Use the "Day" slider (or Next/Prev buttons) to travel through time. You will see the total numbers update for Susceptible, Exposed, Infectious, and Recovered agents for that specific day.
  • Explore the Maps:
    • SEIR Map: Shows the concentration of infected people across different neighborhoods on the map.
    • Reporting Map: Shows which neighborhoods actually reported being sick (simulating real-world testing gaps and reluctance).
  • Analyze the Chart: The line chart at the bottom shows the classic pandemic "curve" over the entire simulation period.
