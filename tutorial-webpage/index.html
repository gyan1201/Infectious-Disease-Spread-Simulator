Building the LLM Disease Simulator

A step-by-step technical guide to reproducing the "An Infectious Disease Spread Simulation Based on Large Language Model Decision Making" framework.

About the Project

This project implements an Agent-Based Model (ABM) that simulates infectious disease spread using Large Language Models (LLMs) to dictate agent reporting behaviors. Built with GeoMason (Java) for spatial simulation and Streamlit (Python) for visualization.

Core Components:

  • vanilla-0.1-jar-with-dependencies.jar: The Java MASON backend engine simulating Atlanta.
  • DecisionBank.csv: The lookup table acting as the LLM's behavioral brain.
  • app.py: The Streamlit GUI for analyzing output graphs and geographical spread.

1 Prerequisites

Before you begin, ensure your system has the following installed:

  • Java 17+ (Required to compile and run MASON)
  • Apache Maven 3.9+ (For building the Java project)
  • Conda / Anaconda (For isolating Python dependencies)
  • Git

2 Clone and Setup MASON Dependencies

The project relies on specific versions of MASON and GeoMason that aren't available on Maven Central. They must be installed locally.

git clone https://github.com/cruiseresearchgroup/disease-simulator-LLM_agent.git
cd disease-simulator-LLM_agent

# Install bundled JARs to your local .m2 repository
mvn install:install-file -Dfile=libs/jts-1.13.jar -DgroupId=com.vividsolutions -DartifactId=jts -Dversion=1.13 -Dpackaging=jar
mvn install:install-file -Dfile=libs/geomason-1.5.jar -DgroupId=edu.gmu.cs -DartifactId=geomason -Dversion=1.5 -Dpackaging=jar
mvn install:install-file -Dfile=libs/mason-19.jar -DgroupId=edu.gmu.cs -DartifactId=mason -Dversion=19 -Dpackaging=jar
mvn install:install-file -Dfile=libs/mason-tools-1.0.jar -DgroupId=edu.gmu.cs -DartifactId=mason-tools -Dversion=1.0 -Dpackaging=jar

3 Compile the Java Backend

Build the "fat JAR" containing all dependencies using the Maven Assembly Plugin.

# Compile the simulation engine
mvn assembly:single

This creates the executable file: target/vanilla-0.1-jar-with-dependencies.jar.

4 Run the Simulation

The simulation uses run.bat (Windows) or run.sh (Mac/Linux).

Windows Path Warning GeoMason throws a ShapeFileImporter error if your project path contains spaces (e.g., "NEW PROJECTS"). If you encounter this, use the subst workaround below to map the folder to a virtual drive letter. 
# Fix spaces in path (Windows Only)
subst X: "C:\path\to\disease-simulator-LLM_agent"
X:

# Run a 30-day simulation with prefix "atl-test"
.\run.bat atl-test 30

Outputs are generated in logs/atl-test/, including DiseaseReports.tsv and spatial graphs.

5 Set Up the Python GUI Environment

Create a fresh Conda environment for the Streamlit dashboard.

conda create -n disease-sim-gui python=3.10 geopandas pyproj geopy numpy pandas scipy scikit-learn -c conda-forge -y
conda activate disease-sim-gui

# Install UI specific packages
pip install "streamlit>=1.18.0" folium streamlit-folium "altair<5" overpass

6 Launch the Interactive Dashboard

Start the Streamlit server to visualize your generated simulation data.

# Ensure you are still in the project root directory
streamlit run gui/app.py

The dashboard will automatically open in your browser at http://localhost:8501.
